# 在线对战跳跳棋实现方案

本方案面向快速落地：先实现 2 人对战，架构上可扩展到 4/6 人；不做观战；加入表情包功能。

## 技术栈

- 前端：React + Vite + TypeScript
- 后端：Node.js + Socket.IO
- 状态：服务端权威内存房间（后续可接 Redis）
- 部署：先本地运行，后续可部署到任意云环境

## 关键设计

- 权威状态：服务端保存棋盘与回合，客户端仅发送“意图”
- 实时同步：Socket.IO 事件广播到房间
- 断线重连：重连后拉取房间快照并恢复
- 防作弊：服务端校验合法走子与回合顺序
- 表情包：内置表情列表，事件广播与展示

## 里程碑概览

- 里程碑 1：规则引擎 + 本地对战
- 里程碑 2：在线对战 + 房间
- 里程碑 3：断线重连 + 稳定性
- 里程碑 4：基础防作弊 + 表情包
- 里程碑 5：部署与文档

## 扩展到 4/6 人准备

- 棋盘与初始位置配置化（人数 -> 初始点位）
- 玩家座位与回合顺序列表化
- UI 玩家槽位渲染与状态显示
