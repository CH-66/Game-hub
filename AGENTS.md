# 开发规范（Game-hub / 跳跳棋在线对战项目）

本规范适用于当前仓库所有代码与文档，尤其是客户端与服务端的协作边界。

## 1. 技术栈与目录约定

- 前端：React + Vite + TypeScript
- 后端：Node.js + Socket.IO
- 目录建议：
  - `client/` 前端工程
  - `server/` 后端工程
  - `shared/` 前后端共享类型与协议
  - `doc/` 里程碑与方案文档

## 2. 编码风格

- TypeScript 全量启用，避免 `any`
- 文件命名：`kebab-case`；组件使用 `PascalCase.tsx`
- 变量/函数：`camelCase`
- 常量：`UPPER_SNAKE_CASE`
- 使用显式类型与接口（尤其是网络协议与状态）

## 3. 状态与同步原则

- 服务端权威：客户端仅发送“操作意图”
- 服务端校验合法性并广播结果
- 房间状态必须是单一可信来源
- 断线重连以服务端快照为准

## 4. Socket.IO 事件规范

- 事件命名统一：`domain:action`（如 `room:create`、`move:intent`）
- 所有事件 payload 必须有类型定义（放在 `shared/`）
- 客户端与服务端必须校验 payload 结构

## 5. 防作弊与安全基线

- 只接受合法回合的玩家指令
- 对非法请求做拒绝与记录
- 对重复/高频请求进行节流

## 6. 文档与里程碑约束

- `doc/` 中每个里程碑文件必须包含 Todo List
- 每完成一个 Todo，需要及时更新为已完成状态
- 每个里程碑结束时必须更新验收结果

## 7. 测试与验证

- 规则引擎要有最小单测覆盖：合法移动、多跳、胜负
- 断线重连与回合锁需包含手动验证步骤说明

## 8. 扩展到 4/6 人的要求

- 初始化棋子布局与胜负判定必须可配置
- 回合顺序以列表驱动，不能写死 2 人逻辑
